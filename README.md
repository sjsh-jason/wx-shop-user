# 餐饮会员小程序

为快餐/外卖餐饮企业打造的微信小程序，帮助商家留住老客户、提升用户复购率。

## 项目简介

本项目包含微信小程序前端和 Java 后端服务，支持 C 端会员和 B 端商家两种角色：
- **C 端会员**：扫码注册、促销抢购预约、积分商城兑换、幸运抽奖、每日签到
- **B 端商家**：扫码核销、促销商品管理、积分规则配置、抽奖管理、客户列表管理

## 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | 原生微信小程序 |
| 后端 | Spring Boot 2.7.18、Java 11 |
| ORM | MyBatis-Plus 3.5.3 |
| 数据库 | MySQL 8.0 |
| 安全 | JWT 0.11.5 |
| 工具 | Hutool 5.8.20、Lombok |

## 后端快速开始

### 环境要求
- JDK 11+
- Maven 3.6+
- MySQL 8.0+

### 本地启动

1. 创建数据库并导入初始化脚本（如有）
2. 修改 `backend/src/main/resources/application.yml` 中的数据库连接配置
3. 执行以下命令启动服务：

```bash
cd backend
mvn spring-boot:run
```

或打包后运行：

```bash
cd backend
mvn clean package
java -jar target/member-1.0.0.jar
```

## 小程序快速开始

1. 使用微信开发者工具打开 `miniprogram` 目录
2. 在开发者工具中填入自己的 AppID
3. 修改 `app.js` 中的后端接口地址
4. 点击编译预览

## 主要功能

### C 端会员
- 扫码注册登录
- 促销产品抢购预约
- 积分商城兑换
- 幸运抽奖（九宫格）
- 每日签到
- 积分明细查询

### B 端商家
- 手机号白名单权限控制
- 扫码核销（预约 / 兑换）
- 促销商品上下架管理
- 积分规则配置
- 幸运抽奖奖品池管理
- 客户列表与消费记录

## 开发规范

- 函数使用小驼峰命名
- 组件使用大驼峰命名
- 常量使用全大写下划线分隔
- 代码注释使用中文
- 优先使用函数式组件
- 每个 PR 只解决一个问题
- 代码需包含单元测试
